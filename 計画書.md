# 翻訳補助アプリ 計画書

## 1. 目的

Zoomで英語圏の人と会議をする際の、言語の壁を下げるための補助ツール。
有料の翻訳ツールに頼らず、ブラウザ標準機能＋外部APIで実現する。

## 2. 全体ゴール（最終形）

| 機能 | 内容 | 開発フェーズ |
|------|------|------------|
| スピーキング補助 | 自分の日本語 → 英語に翻訳・表示・読み上げ | **フェーズ1（完了）** |
| リスニング補助 | 相手の英語 → 日本語字幕で表示・読み上げ | **フェーズ2（完了）** |

---

## フェーズ1：スピーキング補助

### 3. スコープ

#### やること
- マイクから日本語を音声認識（聞き取り）
- 認識した日本語を英語へ自動翻訳
- 翻訳結果を画面に大きく表示
- 翻訳結果を英語音声で読み上げ
- 認識テキストの手動編集に対応
- 直近のやり取りを履歴として保持

#### やらないこと（フェーズ1では対象外）
- 相手の英語音声の取り込み（フェーズ2）
- 翻訳結果のZoomへの自動入力
- 会話全体の議事録保存・エクスポート
- ログイン・アカウント機能

### 4. 技術構成

| 役割 | 使用技術 | 料金 | 備考 |
|------|---------|------|------|
| 音声認識（日→文字） | Web Speech API (`webkitSpeechRecognition`) | 無料 | Chrome必須 |
| 翻訳（日→英） | MyMemory翻訳API | 無料 | 匿名で1日上限あり |
| 音声合成（英読み上げ） | Web Speech API (`SpeechSynthesis`) | 無料 | キー不要 |
| 実行環境 | 単一HTMLファイル（Chromeで開くだけ） | 無料 | サーバー不要 |

### 5. 処理フロー

```
[マイク] 日本語で話す
   ↓ Web Speech API
日本語テキスト（画面表示・編集可）
   ↓ MyMemory API
英語テキスト（画面に大きく表示）
   ↓ SpeechSynthesis
英語の読み上げ音声
   ↓
履歴に追加
```

### 6. ファイル

```
speaking-helper.html  ← フェーズ1アプリ本体（単一ファイル）
```

### 7. 受け入れ基準（完了）

- [x] Chromeで開き、マイク許可後に日本語を認識できる

- [x] 翻訳結果が英語で読み上げられる
- [x] テキストを手動編集して翻訳し直せる

## フェーズ2：リスニング補助

### 8. スコープ

#### やること
- PCのシステム音声（Zoom・YouTube等）をブラウザで直接キャプチャ
- 英語音声をリアルタイムで文字起こし（OpenAI Whisper API）
- 文字起こし結果を日本語に翻訳（GPT-4o-mini）
- 日本語字幕を画面に大きく表示
- 翻訳した日本語を音声で読み上げ（Web Speech API）
- 翻訳履歴の保持

#### やらないこと
- 翻訳結果のZoomへの自動入力
- 会話全体の議事録エクスポート
- ログイン・アカウント機能

### 9. 技術構成

| 役割 | 使用技術 | 料金 | 備考 |
|------|---------|------|------|
| システム音声キャプチャ | `getDisplayMedia()` + ScriptProcessor | 無料 | Chrome必須・画面共有ダイアログが開く |
| PCM音声取得 | ScriptProcessor（Web Audio API） | 無料 | 16kHz モノラル Float32 を直接取得 |
| VAD（発話区間検出） | RMSベースの無音検出 | 無料 | 話の切れ目で自動チャンク分割 |
| WAV変換 | 自前実装（pcmToWav関数） | 無料 | ライブラリ不要 |
| 音声認識（英→文字） | OpenAI Whisper API（whisper-1） | $0.006/分 | APIキー必要 |
| 翻訳（英→日） | OpenAI GPT-4o-mini | 約$0.0001/回 | 同じAPIキーで利用可 |
| 翻訳フォールバック | Google翻訳（非公式API） | 無料 | GPT失敗時に自動切替 |
| 日本語読み上げ | Web Speech API（SpeechSynthesis） | 無料 | 速度1.5x/2x/2.5xから選択 |
| 実行環境 | 単一HTMLファイル（Chromeで開くだけ） | 無料 | サーバー不要 |

### 10. 処理フロー

```
[画面共有ダイアログ] 「画面全体」+「音声を共有」を選択
   ↓ getDisplayMedia()
システム音声ストリーム（Zoom・YouTube等）
   ↓ ScriptProcessor（16kHz PCM）
VAD：無音検出でチャンクを自動分割（最大3〜8秒）
   ↓ pcmToWav()
WAVファイル（16kHz / 16bit / モノラル）
   ↓ OpenAI Whisper API（whisper-1）
英語テキスト（ハルシネーション検出でフィルタリング）
   ↓ 文末・語数で翻訳タイミングを判断
   ↓ GPT-4o-mini（ビジネス会議向けプロンプト）
日本語テキスト
   ↓
字幕表示 + 履歴追加 + 読み上げ（Web Speech API）
```

### 11. 主な実装上の工夫

| 課題 | 解決策 |
|------|--------|
| MediaRecorderのWebMデコード失敗 | ScriptProcessorでPCMを直接取得し、デコード工程を廃止 |
| 話の途中でのチャンク切断 | VAD（RMS無音検出）で発話の切れ目に合わせて自動分割 |
| 翻訳が文中途中で出る | 文末（.!?）検出または15語超で翻訳バッファをフラッシュ |
| Whisperのハルシネーション | 繰り返しパターン検出で幻覚出力を破棄 |
| 字幕と英語テキストの不一致 | 翻訳完了と同時に英語・日本語を同期更新 |
| 翻訳サービス障害 | GPT-4o-mini → Google翻訳 の自動フォールバック |
| 画面レイアウトの上下ジャンプ | `display:none` を `visibility:hidden` に変更 |

## 15. 制約・共通事項

- **Chrome必須**: `getDisplayMedia()` / Web Speech API はChrome系ブラウザのみ安定動作
- **APIキー管理**: OpenAI APIキーはページ内メモリのみに保存（再読込で入力が必要）
- **翻訳精度**: 短い発話・専門用語・訛りは精度が落ちる場合がある
- **コスト**: OpenAI APIは有料（利用者各自のアカウントが必要）
