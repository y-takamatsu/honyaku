# 翻訳補助アプリ フェーズ2 計画書 — リスニング補助

## 1. 目的

Zoom会議で相手の英語を聞きながら、リアルタイムで日本語の字幕として画面に表示し、内容理解を助ける。

## 2. フェーズ2のスコープ

### やること
- Zoomの相手の英語音声を取り込み
- 英語を文字起こし（音声→英文）
- 英文を日本語に翻訳
- 日本語字幕として画面に大きく表示
- 直近のやり取りを履歴として保持
- 文字起こし結果（英語）と翻訳（日本語）を並べて表示

### やらないこと
- 自動議事録エクスポート（必要なら別フェーズで）
- 英語以外の言語対応
- スピーカー識別（誰が話したか）

## 3. 最大の技術課題

> **Zoomで相手が話している英語の音声を、どうやってブラウザのアプリに取り込むか**

フェーズ1で使った Web Speech API は「マイクから入ってきた音」しか聞き取れない。Zoomから流れてくる音はPCのスピーカーから出るだけで、ブラウザの音声認識は拾えない。これがフェーズ2の核心的な課題。

## 4. アプローチ候補

| 案 | 概要 | コード量 | ユーザー作業 | 認識速度 | 認識精度 |
|----|------|---------|-------------|---------|---------|
| **A** | 仮想オーディオ + Web Speech API | 少 | VB-CABLE等の導入が必要 | 速い（リアルタイム） | 高い |
| **B** | ブラウザのタブ音声キャプチャ + Whisper.js | 多 | なし | 遅め（数秒〜） | 中〜高 |

### Option A: 仮想オーディオ方式（推奨）

**仕組み**
- 「VB-CABLE」（無料Windowsアプリ）でZoomの音声をマイク入力に流し込む
- フェーズ1と同じ Web Speech API で、今度は英語(`en-US`)を聞き取り
- 文字起こしを MyMemory APIで英→日翻訳
- 結果を画面に字幕表示

**メリット**
- フェーズ1のコードをほぼ流用できる（軽い変更）
- 認識がリアルタイム・高精度（Google系の音声認識）

**デメリット**
- ユーザーが VB-CABLE と VoiceMeeter Banana のインストール・初期設定が必要（1回のみ）
- 同時に「自分はZoomを聞ける状態」を保つルーティング設定が必要

### Option B: ブラウザ内完結方式（将来案）

**仕組み**
- Zoom を Web版で開き、Chromeのタブ音声をキャプチャ（`getDisplayMedia`）
- ローカルで動く Whisper.js（transformers.js + Hugging Face）で文字起こし
- 翻訳・表示はOption Aと同じ

**メリット**
- ユーザーは追加インストール不要、画面の指示通りクリックするだけ

**デメリット**
- 初回にWhisperモデル（~100MB）をダウンロード
- PCスペックによっては動作が重い・遅延が出る
- コードが大幅に増える

## 5. 推奨方針

**Option A を採用**。理由:

1. 既存コードを活用でき、短期間で動くものが作れる
2. 認識のリアルタイム性が高く、会議用途に向いている
3. VB-CABLE等の導入は一度きりで、慣れれば負担は小さい

Option B は将来的な選択肢として温存（より楽な操作感を求めるとき、または社用PCで仮想オーディオを入れられない場合）。

## 6. 技術構成（Option A）

| 役割 | 使用技術 | 料金 | 備考 |
|------|---------|------|------|
| 音声入力ルーティング | **VB-CABLE** + **VoiceMeeter Banana** | 無料 | 1回のみインストール |
| 音声認識（英→英文） | Web Speech API (`webkitSpeechRecognition`, `lang=en-US`) | 無料 | Chrome必須 |
| 翻訳（英→日） | MyMemory翻訳API | 無料 | 上限はフェーズ1と同様 |
| 字幕表示 | HTML/CSS | 無料 | 大きめの文字、自動スクロール |

## 7. 処理フロー

```
[Zoom] 相手が英語で話す
   ↓ VoiceMeeter Banana で音声を分岐
   ├─→ あなたのヘッドホン（聞こえる）
   └─→ VB-CABLE（仮想マイク入力）
          ↓ Web Speech API (en-US)
       英文（画面に表示）
          ↓ MyMemory API (en→ja)
       日本語字幕（画面に大きく表示）
          ↓
       履歴に追加
```

## 8. ユーザー側の事前準備（Option A）

1. [VB-CABLE](https://vb-audio.com/Cable/) をダウンロード・インストール（無料）
2. [VoiceMeeter Banana](https://vb-audio.com/Voicemeeter/banana.htm) をダウンロード・インストール（無料）
3. PC再起動
4. 音声ルーティングを設定（手順書はアプリ内のヘルプに記載予定）
   - Zoomの出力 → VoiceMeeter Input
   - VoiceMeeter → 自分のヘッドホン (A1) ＋ VB-CABLE Input (B1)
   - Chromeのマイク = CABLE Output

> ⚠️ この準備が手間に感じる場合は、後日 Option B（タブキャプチャ + Whisper.js）の検討もできる。

## 9. 画面構成

1. 開始/停止ボタン（マイク=VB-CABLEを聞き始める）
2. 状態表示（接続中・聞き取り中など）
3. 英文表示エリア（小さめ・薄い色）
4. 日本語字幕エリア（**大きく・読みやすく**）

6. 設定（自動翻訳ON/OFF、字幕フォントサイズ、メール登録）

## 13. 制約・既知のリスク

- **Windows前提**: VB-CABLE は Windows向け（Mac版もあるが構成が違う）
- **音声ルーティング設定の難しさ**: ユーザー側の初期設定がフェーズ1より重い
- **翻訳上限**: リスニングは消費語数が多くなりがち → メール登録ほぼ必須
- **専門用語の認識精度**: 一般会話は強いが固有名詞・専門用語は弱い
- **Web Speech APIの英語安定性**: 長時間連続認識は途切れることがある → 自動再開する仕組みが必要
