# 翻訳補助アプリ フェーズ2 計画書 — リスニング補助（改訂版）

> 📝 改訂: 当初Option A（仮想オーディオ）を採用していたが、ユーザー側の準備が重いため **Option B（ブラウザ完結型）** に方針変更。

## 1. 目的

Zoom会議で相手の英語を聞きながら、リアルタイムで日本語の字幕として画面に表示し、内容理解を助ける。

## 2. フェーズ2のスコープ

### やること
- Zoomの音声をブラウザの画面共有機能で取り込み
- 英語を文字起こし（音声→英文）
- 英文を日本語に翻訳
- 日本語字幕として画面に大きく表示
- 直近のやり取りを履歴として保持

### やらないこと
- 自動議事録エクスポート
- 英語以外の言語対応
- スピーカー識別

## 3. 技術上の核心課題と解決方針

**課題**: Zoomで相手が話している英語の音声を、どうやってブラウザのアプリに取り込むか。

**解決（Option B方式）**:
Chromeの「画面共有」機能（`getDisplayMedia`）には「コンピューターの音声を共有」オプションがあり、これを使うと **PC全体の音声を MediaStream として取得できる**。これをブラウザ内のAI（Whisper.js）に渡して文字起こしする。

→ **追加ソフトのインストール不要。Windowsの音声設定変更も不要。**

## 4. 採用アプローチ: Option B（ブラウザ完結型）

### 仕組み
1. ユーザーが「聞き取り開始」ボタンを押す
2. Chromeの画面共有ダイアログで「画面全体」+「音声を共有」を選択
3. PC全体の音声がアプリに流れ込む
4. ブラウザ内で動く Whisper（OpenAIの音声認識AI）で文字起こし
5. 英文を MyMemory API で日本語に翻訳
6. 字幕として大きく表示

### メリット
- **追加ソフトなし**: VoiceMeeter等のインストール不要
- **Windows/Mac両対応**
- **設定変更なし**: Zoomもデスクトップアプリのまま使える
- **Whisperは固有名詞・専門用語に強い**

### デメリット
- 初回のみAIモデルのダウンロード（~40〜150MB、ブラウザにキャッシュされる）
- 認識に2〜5秒の遅延が発生
- 古いPCでは動作が重くなる可能性

### Option A（廃案）
仮想オーディオ（VoiceMeeter等）を使う方式は、ユーザーの初期設定が重すぎるため不採用。フェーズ2計画書の旧版および Git履歴に残してあるので、必要なら参照可能。

## 5. 技術構成

| 役割 | 使用技術 | 料金 | 備考 |
|------|---------|------|------|
| 音声取り込み | `navigator.mediaDevices.getDisplayMedia()` | 無料 | ブラウザ標準API |
| 音声録音 | `MediaRecorder` API | 無料 | ブラウザ標準API |
| 音声認識 | **Whisper.js**（transformers.js経由） | 無料 | 初回モデルDLあり |
| 翻訳（英→日） | MyMemory翻訳API | 無料 | フェーズ1と同じ |
| 字幕表示 | HTML/CSS | 無料 | 大きめ・サイズ調整可 |
| 実行環境 | 単一HTMLファイル | 無料 | Chromeで開くだけ |

## 6. 処理フロー

```
[ユーザー] 聞き取り開始ボタンをクリック
   ↓
[Chrome] 画面共有ダイアログ（画面全体+音声を共有）
   ↓ MediaStream
[アプリ] 5秒ごとにチャンク録音（MediaRecorder）
   ↓ WebM/Opus → デコード → Float32 PCM
[Whisper.js] 音声 → 英文
   ↓
[MyMemory API] 英文 → 日本語
   ↓
[画面] 字幕として大きく表示 + 履歴に追加
```

## 7. 使うAIモデル

| モデル | サイズ（量子化版） | 速度 | 精度 |
|--------|------------------|------|------|
| **Xenova/whisper-base.en**（採用） | ~80MB | 中 | 高（クリアな英語に強い） |
| Xenova/whisper-tiny.en | ~40MB | 速 | 中 |
| Xenova/whisper-small.en | ~250MB | 遅 | 最高 |

英語専用モデル（`.en`）を使うことで、汎用モデルより精度が上がる。

## 8. ユーザー側の作業（Option B）

### 初回のみ
- なし。アプリを開いて「聞き取り開始」を押すだけ。
- 初回起動時にAIモデルが自動ダウンロード（バックグラウンド、1〜2分）

### 毎回
1. `listening-helper.html` をChromeで開く
2. 「🎧 聞き取り開始」をクリック

4. 自動で字幕表示が始まる

## 13. 制約・既知のリスク

- **Chrome必須**（Edgeもおそらく可、Firefoxは `getDisplayMedia` の音声サポートが限定的）
- **初回モデルダウンロード**: ネットワーク次第で1〜2分かかる
- **PC性能依存**: 古いPCだと文字起こしが遅延しやすい
- **画面共有の音声チェック忘れ**: 「音声を共有」を忘れると動かない → UIで明示
- **MyMemory翻訳上限**: リスニングは消費が早い → メール登録推奨
- **file://プロトコルでのモジュール読込**: 通常は問題ないが、Chromeのバージョン・設定によってはCDNからのES Module読込がブロックされる可能性 → その場合は簡易HTTPサーバー経由で開く（後述）
